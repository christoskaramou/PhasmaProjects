# PhasmaSpace

A Space Engine-style solar system demo for [PhasmaEngine](../../PhasmaEngine): the planets
through Pluto, 459 JPL-cataloged planetary satellites, Saturn's rings, Kuiper Belt and
Oort Cloud guide markers, Earth city lights, and a drifting cloud layer — driven by JPL
ephemerides, NASA-derived surface textures, and an ESA Gaia all-sky background, built
entirely as Lua + assets on the stock engine.

## Requirements

- **PhasmaEngine** — a built or released copy with `PhasmaEditor.exe` / `PhasmaPlayer.exe`.
  Build from source or download a release from the PhasmaEngine repository.

## Run

Everything needed to run is tracked in this project directory; no asset fetch or build step.

1. Clone the project collection next to (or anywhere near) your PhasmaEngine build:
   ```
   git clone https://github.com/christoskaramou/PhasmaProjects
   ```
2. Point the engine at it. In the engine build output dir (next to `PhasmaEditor.exe`),
   create `phasma_settings.json` with the path to this project. The path may be **relative
   to the engine executable** or absolute:
   ```json
   { "project_path": "../../../PhasmaProjects/PhasmaSpace" }
   ```
3. Launch `PhasmaEditor.exe` (or `PhasmaPlayer.exe`) — the startup scene
   (`Assets/Scenes/solar_system.pescene`) loads automatically.

> **Updating engine assets:** After pulling a newer PhasmaEngine build, re-copy
> `Shaders/` and `PassInfo/` from the build's `Assets/` into this project's `Assets/` to
> keep renderer behaviour in sync with the binary.

## In-game UI

The director builds a runtime control panel (works in the editor viewport and
PhasmaPlayer): **orbit-lines toggle**, **auto-exposure toggle**, **follow-orbit
toggle**, **time controls** (pause, /2, x2), and a curated **follow dropdown** for
the Sun, planets, Pluto, Mars/Pluto moons, and the largest major moons. Moon entries
show their parent planet, e.g. `Jupiter / Io`. A top-left HUD
shows camera speed and follow zoom; **mouse wheel** zooms the followed body or changes
free-camera speed (m/s, scaling to km/s / Mm/s / multiples of c). Orbit lines are
sampled from the ephemerides into Lua-owned vertex buffers and drawn through the
script-created manual render pass.

The `SolarSystem` root node carries the director script. Its exposed variables (editor
Properties panel, or `node:get_exposed()` from Lua) drive the simulation:

| Exposed var | Default | Meaning |
|---|---|---|
| `time_scale` | 1/86400 | simulated days per real second (0 freezes) |
| `epoch_jd` | current UTC | Julian date at sim start |
| `animate_in_editor` | true | tick the sim outside play mode |
| `follow` | "Earth" | body the camera tracks (`Sun`, planets, Pluto, and curated moons; "" = free cam) |
| `follow_distance` | 180.0 | camera distance in body radii |
| `follow_orbit` | true | RMB/MMB drag orbits around the followed body |

While following, you can still orbit/zoom by hand: RMB/MMB drag rotates around the
tracked body, and mouse wheel adjusts distance. Follow mode rebases the
dynamic planet nodes around the tracked body so the subject and camera render near
world origin instead of at large heliocentric coordinates; free camera mode also
recenters when the camera drifts far from origin.

## Regenerating the scene

`Assets/Scenes/solar_system.pescene` is generated by running
`Assets/Scripts/solar/build_solar_system.lua` in a project-loaded editor (it is
idempotent and saves the scene). Run it through the editor console or its MCP
`execute_lua` endpoint after editing `planets.lua`, `moon_catalog.lua`, or
`ephemeris.lua`.

## Engine support assets

`Assets/{Fonts,Icons,Materials,Objects,Particles,PassInfo,Shaders,SplashScreen}` are
copies of PhasmaEngine's built-in assets (a project's `Assets/` fully replaces the
engine's `Path::Assets`). **After pulling engine changes, re-copy at least `Shaders/`
and `PassInfo/`** from `PhasmaEngine/build-*/Release/Assets/` or renderer behavior may
drift from the engine binary.

## Scale model

| Constant | Value |
|---|---|
| 1 unit | 10,000 km (distances) |
| Planet/moon/sun radii | **×1 — true scale** |
| 1 AU | 14,959.79 units |

Planet/moon sizes, distances, and orbits are at real scale. Bodies are sub-pixel
dots from any distance (that is what the solar system actually looks like); use the
clickable body markers and the follow camera to navigate. The Kuiper Belt guide rings
use their real 30-50 AU scale; the Oort Cloud rings are compressed visual markers.

Float32 precision at Neptune's orbit (~450k units) is ~0.03 units ≈ 300 km effective,
which is visible when the camera sits only a few body radii from the followed target.
The director therefore keeps planet, moon, shift, and orbit-sample positions as Lua
numbers until the final engine write, then uses a lightweight floating origin so the
visible scene avoids large float cancellations. Orbit guide lines are anchored at
the current epoch, sampled from a sagitta budget, and rebuilt only during scene
binding; the orbit root follows the floating origin so scripts do not mutate orbit
geometry during ticks.

## Data sources & licenses

- **Planet ephemerides:** E.M. Standish (JPL), [Keplerian Elements for Approximate Positions
  of the Major Planets](https://ssd.jpl.nasa.gov/planets/approx_pos.html) (Table 1,
  1800–2050 AD), plus JPL Horizons osculating elements for Pluto.
- **Moon catalog:** JPL Solar System Dynamics [mean satellite elements](https://ssd.jpl.nasa.gov/sats/elem/)
  and [physical parameters](https://ssd.jpl.nasa.gov/sats/phys_par/).
- **Planet textures:** [Solar System Scope](https://www.solarsystemscope.com/textures/)
  (CC BY 4.0, NASA-data derived).
- **Star map:** ESA/Gaia/DPAC [Gaia's sky in colour, equirectangular projection](https://sci.esa.int/web/gaia/-/60196-gaia-s-sky-in-colour-equirectangular-projection)
  (CC BY-SA 3.0 IGO; based on nearly 1.7B Gaia DR2 stars).
- **Physical constants:** NASA NSSDC planetary fact sheets.

Sunlight uses **physically correct inverse-square falloff** (engine setting
`physical_point_falloff`, serialized in the scene; sun intensity = luminance x
distance^2, anchored so Earth at 1 AU is correctly exposed). The director adds
**photographic auto-exposure** (exposed `auto_exposure`, default on): while following
a body, the global light multiplier scales with its solar distance squared, so
Mercury, Earth, and Neptune all read properly exposed — the physics stays real, only
the "camera" adapts. Turn `auto_exposure` off to see the raw gradient (Mercury blown
white, Neptune near-black). The follow camera keeps its offset in the body's rotating
solar frame, so the phase angle you choose stays put while the planet sweeps its
orbit; drag the camera to pick a new angle.

Known demo simplifications: axial tilt applied about the orbital X axis (not true pole
RA/Dec), satellite orbits use JPL mean elements rather than full numerical integration,
the Oort Cloud is represented with compressed visual markers, and there are no
inter-planet shadows. Earth city lights use the engine's `night_emissive` material flag
(emissive fades where direct sunlight is strong, so they only show on the night side).
Moons without texture assets use flat color tints.
